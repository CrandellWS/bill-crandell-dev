6:T1b61,<h2>Summary</h2>
<p>Today I set up a Docker-based infrastructure for running AI web agents with Claude Code. The goal was simple: create a containerized environment where Claude Code starts automatically with specific flags, ready for autonomous web automation tasks. The result is a clean, reproducible development environment that can spin up AI agents on demand.</p>
<h2>What I Built</h2>
<h3>AI Web Agents Docker Setup</h3>
<p><strong>The Infrastructure:</strong></p>
<ul>
<li>Dockerfile with Node.js 20 environment</li>
<li>Claude Code CLI installation and configuration</li>
<li>Auto-start with <code>-r --dangerously-skip-permissions</code> flags</li>
<li>Docker Compose orchestration with persistent volumes</li>
<li>Environment-based API key management</li>
<li>Workspace and config persistence</li>
</ul>
<p><strong>Files Created:</strong></p>
<pre><code>ai_web_agents/
├── Dockerfile              # Container image with Claude Code
├── docker-compose.yml      # Service orchestration
├── .env.example           # API key template
└── README.md              # Setup documentation
</code></pre>
<h3>The Architecture</h3>
<p><strong>Container Setup:</strong></p>
<ul>
<li>Base: Node.js 20 slim image</li>
<li>System deps: curl, git, python3, sudo</li>
<li>Non-root user for security</li>
<li>Global Claude Code CLI installation</li>
<li>Persistent volumes for workspace and config</li>
</ul>
<p><strong>Docker Compose Features:</strong></p>
<ul>
<li>Interactive TTY support for Claude Code sessions</li>
<li>Environment variable injection (API keys)</li>
<li>Volume mounts for data persistence</li>
<li>Bridge networking</li>
<li>Auto-restart policy</li>
</ul>
<h3>Key Implementation Details</h3>
<p><strong>Dockerfile Strategy:</strong></p>
<pre><code class="language-dockerfile"># Node.js 20 slim for minimal footprint
FROM node:20-slim

# System dependencies
RUN apt-get update &#x26;&#x26; apt-get install -y \
    curl git python3 python3-pip sudo

# Non-root user for security
RUN useradd -m -s /bin/bash aiuser

# Global Claude Code CLI
RUN npm install -g @anthropic-ai/claude-code

# Auto-start with specific flags
CMD ["claude", "-r", "--dangerously-skip-permissions"]
</code></pre>
<p><strong>Volume Persistence:</strong></p>
<ul>
<li><code>./workspace</code> → Container workspace (your projects)</li>
<li><code>./config</code> → Claude Code configuration</li>
<li>Ensures work survives container restarts</li>
</ul>
<h2>Key Learnings</h2>
<h3>Container-First Development</h3>
<p>Building AI automation infrastructure with containers from the start. Dockerizing development environments ensures reproducibility across machines and makes deployment trivial.</p>
<h3>Auto-Start Configuration</h3>
<p>Using CMD to automatically launch Claude Code with flags. No manual intervention needed - container starts, Claude Code runs with the right permissions.</p>
<h3>Volume Strategy for AI Tools</h3>
<p>Persistent volumes for both workspace AND config. AI tools often have state (conversation history, learned patterns) that should survive restarts.</p>
<h3>Security with Convenience</h3>
<p>Running as non-root user with sudo access. Balance between security best practices and development flexibility for AI agents that might need system access.</p>
<h2>Technical Deep Dive</h2>
<h3>Why These Flags?</h3>
<p>The <code>-r --dangerously-skip-permissions</code> flags are critical for automated agents:</p>
<ul>
<li><code>-r</code>: Resume mode for continuous operation</li>
<li><code>--dangerously-skip-permissions</code>: Allows autonomous actions without prompts</li>
</ul>
<p>Perfect for web automation scenarios where you want Claude Code to work independently.</p>
<h3>Interactive TTY Setup</h3>
<pre><code class="language-yaml">stdin_open: true
tty: true
</code></pre>
<p>These Docker Compose settings enable interactive sessions. Critical for attaching to the running Claude Code instance and monitoring/controlling it.</p>
<h3>Environment Variable Pattern</h3>
<pre><code class="language-yaml">environment:
  - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
</code></pre>
<p>Reads from <code>.env</code> file, never commits secrets to git. Clean pattern for managing credentials in containerized environments.</p>

<li>Set up environment variables</li>

<p>This project demonstrates infrastructure-as-code thinking for AI tools. Instead of manually configuring development environments, we codify the setup and make it reproducible.</p>
<p>The pattern here applies broadly:</p>
<ul>
<li><strong>Container</strong>: Isolated environment with all dependencies</li>
<li><strong>Orchestration</strong>: Docker Compose for service management</li>
