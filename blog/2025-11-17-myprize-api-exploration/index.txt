<p>Today was all about exploring the MyPrize Creator API to plan the automated prize distribution system for Moonie-v2.</p>
<h2>What We Built</h2>
<p>Created a comprehensive API explorer at <code>/myprize-api-explorer/</code>:</p>
<ul>
<li><strong>Interactive credential tool</strong> - Secure secret key input (only prompts for sensitive data)</li>
<li><strong>HMAC-SHA512 authentication</strong> - Full implementation of MyPrize's signature algorithm</li>
<li><strong>Comprehensive endpoint testing</strong> - Automated tests for 7+ API endpoints</li>
<li><strong>Error handling</strong> - Graceful handling of permission errors and API issues</li>
</ul>
<h2>Key Findings</h2>
<h3>âœ… What Works</h3>
<p>Successfully authenticated and accessed:</p>
<ul>
<li>Referral volume statistics</li>
<li>Referred users list</li>
<li>Scaled volume aggregation</li>
<li>Room activity monitoring</li>
</ul>
<p>Authentication is working perfectly - HMAC-SHA512 signature generation is correct.</p>
<h3>ðŸ”’ Critical Blocker</h3>
<p>The core functionality we need - <strong>promotional prize assignment</strong> - requires additional API permissions:</p>
<pre><code>GET /api/creator/promos/assignable
POST /api/creator/promos/{promo_id}/assign
</code></pre>
<p>Error: <code>apiKeyPermissionDisabled</code></p>
<p><strong>This is the blocker for Moonie-v2 integration.</strong></p>
<h2>Why This Matters</h2>
<p>The entire point of MyPrize integration is to <strong>automatically assign prizes to Moonie winners</strong>:</p>
<ol>
<li>User plays along on MyPrize â†’ qualifies for Moonie spin</li>
<li>Moonie-v2 determines winner and prize tier</li>
<li>Backend calls MyPrize API to assign the prize</li>
<li>Winner gets notified via PubNub chat</li>
<li>Prize appears in their MyPrize account instantly</li>
</ol>
<p>Without promo assignment access, we can't build or test this flow.</p>
<p>The other endpoints (stats, referrals, etc.) are just for website content - not the core automation.</p>

<ul>

<li>Real-time room monitoring (poll every 5 seconds)</li>
<li>Automatic prize fulfillment system</li>
<li>PubNub chat notifications</li>
<li>WebSocket connection to Moonie-v2 frontend</li>
</ul>

<h2>Technical Details</h2>
<h3>Authentication Flow</h3>
<p>MyPrize uses HMAC-SHA512 signatures for all authenticated requests:</p>
<pre><code class="language-javascript">const timestamp = Math.floor(Date.now() / 1000);
const stringToSign = `${timestamp}|${httpMethod}|${path}`;
const signature = createHmac('sha512', secret)
  .update(stringToSign)
  .digest('hex');

const xApiKey = `${apiKeyLabel}|${timestamp}|${signature}`;
</code></pre>
<p>Header: <code>x-api-key: {apiKeyLabel}|{timestamp}|{signature}</code></p>
<h3>Rate Limits</h3>
<ul>
<li><strong>5000 requests per 5 minutes</strong> - More than enough for our use case</li>
<li>Respect cache-control headers to avoid unnecessary requests</li>
</ul>
